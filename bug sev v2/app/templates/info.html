Code Metrics

Metric Description
LC (Lines of Code) The number of lines in the code.
PI (Programming Index) A derived metric combining various aspects of the code's complexity, readability, and maintainability.
MA (Maintainability Index) Evaluates how easy it is to maintain the code. Higher values indicate easier maintenance.
NBD (Nested Block Depth) Measures the maximum depth of nested blocks in the code, indicating complexity.
ML (McCabe's Cyclomatic Complexity) Measures the complexity of a program's control flow by counting linearly independent paths.
D (Design Complexity) Quantifies the complexity of the code's design, including architecture and structure.
MI (Maintainability Index) Evaluates how easy it is to maintain the code. Higher values indicate easier maintenance.
FO (Fan-Out Complexity) Measures the number of other modules or functions a particular module or function calls.
R (Reusability Index) Assesses how reusable the code is. Higher values suggest easier reuse in other projects.
E (Effort Estimate) Estimates the effort required to develop or maintain the code based on various factors.
